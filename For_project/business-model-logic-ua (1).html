🎯 Мета: нульовий залишок
Фази lifecycle артикула
ФАЗА 0

Pre-arrival

Товар замовлений, ще в дорозі. Агент відстежує дату надходження і готує полицю.

ФАЗА 1

Warm-up (прогрів)

Перші 2–6 тижнів. Продажі нижче норми — товар "теплий" сезон ще не настав. Агент не б'є тривогу.

ФАЗА 2

Active selling

Пік сезону. Агент відстежує хіти для дозамовлення та слоу-муверів для корекції.

ФАЗА 3

Pre-sale planning

За 4–6 тижнів до сейлу. Агент розраховує: чи встигнемо продати залишок без знижки?

ФАЗА 4

Sale & clearance

Фінальний розпродаж. Агент рекомендує глибину знижки для досягнення нульового залишку.

Модель 2 — Carryover логіка

Постійний асортимент з дозамовленнями

Годинники D1 Milano, класичні сумки, базові аксесуари — одні й ті самі артикули продаються роками. Мета: безперервний запас без надлишку.

♻️ Модель дозамовлення

Коли агент ініціює дозамовлення

  • Залишок досяг точки перезамовлення (ROP)ROP = avg_daily_sales × lead_time + safety_stock
  • Прогнозний дефіцит через < (lead_time + 7 днів)Буфер на логістичні затримки
  • Постачальник оголосив про підвищення цінАгент рекомендує завчасний запас
  • Артикул накопичив достатньо статистики (>30 продажів)До цього — консервативний прогноз
📊 EOL — кінець життя моделі

Якщо бренд знімає модель

  • Агент відстежує сигнали EOL від клієнта або новини бренда
  • Розраховує "останній оптимальний замовлення"Скільки взяти, щоб продати без залишку
  • Рекомендує не дозамовляти понад прогноз на 6 місяцівРизик зависання після знімання
  • Переключає логіку на clearance при EOL-підтвердженні

System Prompt — структура для двох режимів
🌿 Режим SEASONAL — блок для System Prompt
Клієнт працює в СЕЗОННІЙ моделі. Правила аналізу:

1. Кожен артикул має дату надходження, дату початку активного сезону та дату сейлу
2. Класифікуй фазу кожного SKU: Pre-arrival / Warm-up / Active / Pre-sale / Sale
3. В фазі Warm-up — не рахуй низькі продажі як аномалію. Порівнюй з прогнозом для цієї фази
4. Відстежуй sell-through прогрес: чи встигне артикул продатись до сейлу?
5. Формула: залишок / avg_daily_sales = дні до нуля. Порівняй з датою сейлу
6. Якщо (дні до нуля > дні до сейлу) → рекомендуй промо або ранній сейл
7. Якщо артикул — хіт і дозамовлення можливе → розрахуй optimal reorder qty
8. Кожна нова партія = окрема група артикулів зі своїм lifecycle. Веди їх паралельно.

Мета: 0 залишку сезонного товару після сейлу. Не залишити гроші на полиці.
♻️ Режим CARRYOVER — блок для System Prompt
Клієнт працює в КЕРІОВЕР моделі. Правила аналізу:

1. Артикул не має "кінця сезону" — він продається безперервно
2. Відстежуй ROP (точку перезамовлення): ROP = avg_daily_sales × (lead_time + buffer_days)
3. Buffer для UA логістики: +7 днів до задекларованого lead time постачальника
4. Якщо залишок ≤ ROP → автоматичний алерт на дозамовлення
5. Перед сезонними піками (Лютий, Березень, Жовтень–Листопад) → збільшуй замовлення ×1.3–1.5
6. Якщо артикул не замовлявся 6+ місяців → перевір EOL-ризик у постачальника
7. Якщо оголошено EOL → розрахуй фінальне замовлення на 4–6 місяців і переключи в clearance

Мета: 0 стокаутів. Жоден клієнт не повинен іти без покупки через відсутність товару.
