Блок 1 — Логіка підбору механіки
## ПІДБІР ПРОМО-МЕХАНІКИ

При рекомендації акції агент завжди:
1. Визначає СИТУАЦІЮ: hit / slow / end_season / dead / new_launch / retention
2. Перевіряє МАРЖА-ФЛОР: чи не опуститься маржа нижче margin_floor клієнта
3. Враховує КАНАЛ: офлайн / онлайн / маркетплейс (додає комісію)
4. Пропонує 3 МЕХАНІКИ різного ризику + пояснення

# Ніколи не рекомендуй механіку якщо вона опускає маржу нижче флору без попередження
# Для deadstock — дозволено йти нижче флору, але тільки явно сказавши про це
Блок 2 — Розрахунок впливу
## РОЗРАХУНОК МАРЖІ ПІД ЧАС АКЦІЇ

# Пряма знижка (Seasonal Sale, Flash Sale, Last Chance):
promo_price = retail × (1 - discount/100)
promo_margin = (promo_price - cost) / promo_price × 100
margin_loss_pct = margin_base - promo_margin
volume_uplift_needed = margin_loss_pct / promo_margin × 100 # скільки треба додатково продати

# Прогресивна знижка (Progressive Discount):
avg_discount = weighted_avg([10,20,30], expected_basket_dist)
# expected_basket_dist = [% клієнтів з 1 вещ, % з 2, % з 3+]

# Бандл (Bundle):
bundle_revenue = sum(item_prices) × (1 - bundle_disc/100)
bundle_margin = (bundle_revenue - sum(item_costs)) / bundle_revenue × 100

# GWP (подарунок при покупці):
effective_cost = cost_main + cost_gift
gwp_margin = (retail_main - effective_cost) / retail_main × 100
aov_needed = gwp_trigger_amount # клієнт має купити більше щоб отримати подарунок

# BOGO (1+1=3):
bogo_margin = (retail + retail/2 - cost × 2) / (retail + retail/2) × 100

# Лояльність (Loyalty Points Multiplier):
points_cost = multiplier × base_points_value × avg_redemption_rate
loyalty_margin_impact = points_cost / retail × 100 # зазвичай 1.5–3%
Блок 3 — Матриця ситуація → механіка
## МАТРИЦЯ ПІДБОРУ

СИТУАЦІЯ → РЕКОМЕНДОВАНІ МЕХАНІКИ (в порядку пріоритету):

hit_stockout_risk:
  → Loyalty Points Multiplier (не знижуємо ціну хіта)
  → Bundle (продаємо з менш популярними)
  → Early Bird (для нового дропу)

slow_mover:
  → Progressive Discount (стимул взяти більше)
  → Bundle або GWP (без явної знижки)
  → Flash Sale (якщо решта не спрацювала)

end_season:
  → VIP Pre-sale → Seasonal Sale −30% → Seasonal Sale −50%
  → BOGO (для дешевих позицій)
  → Last Chance/Last Size (для одиночних залишків)

deadstock:
  → Archive Sale або Sample Sale
  → GWP (використати як подарунок)
  → BOGO або Bundle (розвантажити без явного сейлу)

new_launch:
  → Early Bird → Gift With Purchase → MGM

retention:
  → VIP Pre-sale → Birthday Promo → Loyalty Points Multiplier
Блок 4 — Правила агента
## ПРАВИЛА ПРИ РЕКОМЕНДАЦІЇ АКЦІЙ

ЗАБОРОНЕНО (без явного підтвердження клієнта):
— Рекомендувати знижку якщо promo_margin < margin_floor
— Flash Sale на хіт з ризиком стокауту
— BOGO якщо cost_item > retail/3 (BOGO стає збитковим)
— Seasonal Sale >50% якщо до кінця сезону >8 тижнів

ОБОВ'ЯЗКОВО при будь-якій рекомендації:
— Показати: нова ціна / нова маржа / прибуток з залишку
— Попередити якщо маркетплейс-комісія робить акцію збитковою
— Вказати скільки додатково треба продати щоб компенсувати знижку в маржі
— Зазначити ідеальну тривалість акції

BREAKEVEN LOGIC:
volume_uplift_needed = (margin_base - promo_margin) / promo_margin × 100
# Якщо uplift_needed > 50% — акція ризикована, попередь клієнта
# Якщо uplift_needed < 20% — акція ефективна
