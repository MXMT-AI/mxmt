Товарная база (Справочник SKU)
Фундамент — без этого агент не знает, что анализировать
Критично
Обязательные поля
  • Артикул / SKU-кодУникальный идентификатор — основа всех таблиц
  • Название товараПолное и краткое
  • Категория и подкатегорияНапример: Одежда → Куртки → Зимние
  • Бренд / ПоставщикДля анализа по брендам
  • Розничная ценаТекущая цена продажи
  • Закупочная ценаДля расчёта маржи
  • Статус товараАктивный / Архив / Сезонный / Новинка
Дополнительные поля
  • СезонностьЗима / Лето / Всесезон / Конкретные месяцы
  • Целевая аудиторияМужской / Женский / Унисекс / Возраст
  • Страна производства
  • Коллекция / ЛинейкаSpring 2025, Base Collection и т.д.
  • Взаимозаменяемые товарыSKU-аналоги при стокауте
  • Дата ввода в ассортимент

2
История продаж
Основной источник для прогноза и классификации товаров
Критично
Минимальный набор
  • Дата продажи (день)Чем детальнее — тем точнее прогноз
  • SKU проданного товара
  • Количество проданных единиц
  • Сумма продажи (рублей)
  • Канал продажиОфлайн-касса / Сайт / Маркетплейс
Дополнительно
  • Возвраты по SKU и датеВлияет на реальную скорость продаж
  • Продажи со скидкой vs безОрганический vs промо-спрос
  • Точка продажи / склад
  • ID транзакции (для трекинга)
  • Метод оплаты (опционально)
⚠️ Важно — глубина истории

Минимум для запуска агента — 12 месяцев истории продаж. Оптимально — 24–36 месяцев: агент увидит сезонные циклы и сможет строить точные прогнозы. Если история короткая, укажи это явно — агент скорректирует уровень уверенности в прогнозах.

// Пример строки истории продаж (CSV или JSON) { "date": "2025-01-15", "sku": "JKT-ZIM-042", "qty_sold": 3, "revenue": 26700, "channel": "offline", // offline / online / marketplace "is_promo": false, // продажа по акции? "returns": 0 }
3
Остатки на складе (текущие и динамика)
Актуальный снэпшот + движение запасов
Критично
Текущие остатки (на сегодня)
  • SKU
  • Количество единиц на складе
  • Разбивка по складским точкамСклад 1, Склад 2, Торговый зал
  • Зарезервированные единицыПод заказы, но ещё не отгружены
  • Товары в пути (ожидаемые PO)
Параметры пополнения
  • Lead time по каждому поставщикуДни от заказа до получения на складе
  • Минимальный заказ (MOQ)
  • Целевой страховой запасСколько дней запаса считается безопасным
  • Максимальная ёмкость склада по категориям
  • Частота поставок (раз в 2 недели / раз в месяц)
// Снэпшот остатков + параметры пополнения { "sku": "JKT-ZIM-042", "qty_on_hand": 18, "qty_reserved": 3, "qty_available": 15, // on_hand - reserved "qty_in_transit": 24, // в пути, ожидается "lead_time_days": 14, "moq": 12, "safety_stock_days": 7, // целевой резерв в днях "snapshot_date": "2026-04-22" }
4
Бизнес-контекст (о вашем магазине)
Агент должен понимать, как устроен ваш бизнес — это задаётся один раз
Контекст
Формат бизнеса
  • Тип магазинаМонобренд / мультибренд / концепт
  • Количество точек продаж
  • Онлайн + офлайн или только одно
  • Ценовой сегментМасс-маркет / средний / премиум
  • Основная аудитория (пол, возраст)
Ключевые периоды
  • Высокий сезонМесяцы пиковых продаж
  • Низкий сезон
  • Регулярные акцииЧёрная пятница, 8 марта, конец сезона
  • Даты смены коллекций
  • Даты поставок (плановый календарь)
Пороговые значения
  • Порог «хит»+30% продаж за 7 дней — это ваш порог?
  • Порог «слоумувер»Например: -20% за 14 дней
  • Порог стокаут-рискаЗапас меньше X дней = алерт
  • Порог «зависший товар»Нет движения более N дней
  • Минимальная целевая маржа (%)
💡 Подсказка — как передать контекст

Это лучше описать в свободном тексте как часть System Prompt агента. Например: «Мы — мультибренд магазин женской одежды в Москве. Пик продаж — сентябрь–ноябрь и февраль–март. Порог хита — рост продаж более 35% за 7 дней. Критический остаток — менее 10 дней запаса при lead time 14 дней.»

💡 Как передать карту «погода → спрос»

Это экспертное знание, которое вы передаёте агенту один раз текстом: «При температуре ниже 0°C спрос на зимние куртки вырастает на 40–60% относительно нормы. При первом снеге — краткосрочный пик +80% в течение 3 дней. Дождливые дни снижают трафик в офлайн на 20–30%, но не влияют на онлайн.»
