Логіка ціноутворення від RRP

Три моделі розрахунку роздрібної ціни — агент визначає яка застосовується через бриф клієнта

Модель A — від EUR RRP бренду

Основна модель для імпортних брендів
# Крок 1: Переводимо EUR RRP в гривню
rrp_uah = rrp_eur × rate_eur_uah
# Крок 2: Додаємо коефіцієнт (+10% до EUR RRP)
retail_price = rrp_uah × coeff (зазвичай 1.05–1.15)
# Крок 3: Заокруглюємо до прайс-поінту
retail_price → round to price point (990 / 1490 / 1990 / 2990...)

Маржа по категоріях

Gross margin від роздрібної ціни (RRP). Можна порівняти із власними показниками клієнта.

Як читати: Маржа = (RRP − собівартість товару) / RRP × 100%. Не включає операційні витрати (оренда, зарплати, доставка) — це товарна маржа. Markup = 1 / (1 − margin%).

Маркетплейси та їх вплив на маржу

Агент запитує про маркетплейси під час онбордингу та враховує комісію у всіх розрахунках ціни

Логіка агента: Перш ніж рекомендувати ціну чи знижку — агент запитує «Де продаєте цей товар?» і обирає відповідний флор для кожного каналу.

Розрахунок реальної маржі по каналах

На прикладі одного товару: RRP 3,990₴ / Собівартість 1,400₴ / Маржа до комісій 64.9%
Канал Комісія Виручка з продажу Маржа після комісії Мін. ціна продажу (0% маржа) Оцінка каналу
Критично для MDfashion / TSUM: При комісії 30–35% продаж по собівартості 1,400₴ дає ЗБИТОК. Мінімальна ціна в цьому каналі = 1,400 / (1 − 0.32) = 2,059₴, що відповідає знижці −48% від RRP. Агент попереджає якщо клієнт хоче поставити знижку більшу.

Онбординг-запитання агента про канали

Агент задає ці питання при першому запуску або при аналізі конкретного товару
1. Де продаєте цей товар? (можна обрати кілька)
Власний магазин / Власний сайт / MDfashion / Rozetka / Prom.ua / Instagram-продажі / Інший маркетплейс
2. Яка комісія на вашому маркетплейсі? (якщо не стандартна)
Деякі бренди мають індивідуальні умови — агент запитує якщо категорія відрізняється від стандарту.
3. Чи хочете автоматично перераховувати флор-ціну для кожного каналу?
Якщо «Так» — агент показує різні поріги для офлайн, свого сайту і маркетплейсів в одному звіті.
Стратегії переоцінки

3 варіанти + твій сценарій

Агент пропонує три обґрунтовані стратегії переоцінки залежно від ситуації зі стоком, сезоном і каналом продажу

Ці варіанти генеруються агентом під конкретний товар з урахуванням: залишку, днів до сейлу, поточної маржі, наявності маркетплейс-комісії та мінімального маржа-флору клієнта.
Ситуація:
Залишок (шт): Поточна ціна: ₴ Собівартість: ₴

Логіка «Продаж в довісок до основної покупки»

Останні одиниці — як інструмент підвищення середнього чека
Коли використовувати: Залишилось 1–5 шт у непопулярних розмірах (XS або XL). Товар є в наявності, але не продається сам по собі. Ідеально для «допродажу».
  • → Додати до кошика хіта як «рекомендований товар зі знижкою»
  • → При покупці на суму X — отримай цей товар зі знижкою Y%
  • → Бандл: основний товар + довісок = пакет зі знижкою на обидва
  • → Подарункова упаковка — останні шт як преміальний подарунок
# Розрахунок ефективності довіска
addon_price = cost × 1.0 (або нижче для швидкої реалізації)
avg_basket_uplift = addon_price / main_item_price

# Приклад: основний товар 3,990₴
# Довісок по 800₴ (собів. 700₴)
uplift = 800 / 3990 = +20% до середнього чека
margin_addon = (800-700)/800 = 12.5% (краще ніж 0)

Агент рекомендує: якщо залишок ≤3 шт і STR <5% — довісок пріоритетніший за знижку

Блок 1 — Логіка ціноутворення
## ЦІНОУТВОРЕННЯ

МОДЕЛЬ_А (від EUR RRP бренду):
retail_price = rrp_eur × rate_eur_uah × coeff_brand
# coeff_brand: береться з брифу клієнта, за замовч. 1.10
# rate_eur_uah: отримуємо з Minfin API щодня

МОДЕЛЬ_Б (від собівартості):
retail_price = landed_cost × markup
# markup для fashion UA: одяг 2.5–3.0×, аксесуари 3.0–4.0×, взуття 2.5–3.5×

ЗАОКРУГЛЕННЯ до прайс-поінту:

Блок 2 — Маржа і флор
## МАРЖА ТА МІНІМАЛЬНА ЦІНА

# Завжди зберігай для кожного SKU:
landed_cost: собівартість (закупівля + логістика + мито)
margin_floor: мінімальна допустима маржа клієнта (з брифу, за замовч. 20%)
margin_current: (retail - landed_cost) / retail × 100

# Порогові ціни:
price_zero = landed_cost # продаж по собівартості (0% маржа)
price_floor = landed_cost / (1 - margin_floor/100)
price_floor_mp = landed_cost / (1 - margin_floor/100 - mp_commission/100)

max_discount_pct_zero = (retail - price_zero) / retail × 100
max_discount_pct_floor = (retail - price_floor) / retail × 100
max_discount_pct_mp = (retail - price_floor_mp) / retail × 100

# ПРАВИЛО: ніколи не рекомендуй знижку більшу за max_discount_pct_zero
# без явного підтвердження від клієнта ("продаю в збиток свідомо")
Блок 3 — Маркетплейси
## МАРКЕТПЛЕЙС-ЛОГІКА

# Комісії маркетплейсів (оновлюй з брифу клієнта):
mp_commissions = {
  "MDfashion/TSUM": 0.30,
  "Rozetka": 0.12,
  "Prom.ua": 0.08,
  "Instagram/Direct": 0.00,
  "own_site": 0.00,
  "own_store": 0.00
}

# При аналізі переоцінки — ЗАВЖДИ перевіряй:
if client_sells_on_marketplace:
  price_floor_mp = landed_cost / (1 - mp_comm - margin_floor/100)
  if proposed_price < price_floor_mp:
    ALERT: "Ця ціна дає збиток на маркетплейсі після комісії!"
    SHOW: f"Мін. ціна для {mp_name}: {price_floor_mp:.0f}₴"
Блок 4 — Логіка переоцінки 3+1
## ПРОПОЗИЦІЯ ПЕРЕОЦІНКИ

# Агент генерує 3 варіанти + 1 користувацький:

ВАРІАНТ_1 (консервативний): # тримати маржу
  discount = min(15%, max_discount_floor)
  price = retail × (1 - discount)
  new_margin = (price - cost) / price × 100

ВАРІАНТ_2 (агресивний): # швидкий продаж
  if days_to_sale < 21: discount = max_discount_floor
  else: discount = (max_discount_floor + max_discount_zero) / 2

ВАРІАНТ_3 (в 0 або довісок): # ліквідація
  if stock <= 3: рекомендуй довісок до основної покупки
  else: discount = max_discount_zero (продаж по собівартості)

ВАРІАНТ_4 (користувацький):
  Отримати ціль/відсоток від користувача → розрахувати маржу → попередити якщо збиток

# Для кожного варіанту показуй:
# нову ціну, нову маржу, прибуток з одиниці, прибуток з усього залишку
# попередження якщо нижче флору або нижче mp-порогу
Блок 5 — Порівняння з ринком
## БЕНЧМАРК-ПОРІВНЯННЯ

# При аналізі SKU — порівнювати маржу клієнта з ринком:
market_avg = BENCHMARKS[category][subcategory]["avg"]
deviation = margin_current - market_avg
